## 一、Rowboat 交互模式分析

Rowboat 的 Connect Google 流程有以下核心模式：

| 模式 | Rowboat 实现 | 值得借鉴 |
|------|-------------|---------|
| **卡片发现** | ToolkitCard 网格，每张卡显示名称+logo+状态badge | 是 |
| **状态徽章** | Connected(绿)/Not Connected(灰)/Pending(黄) | 是 |
| **Modal 连接流** | 点击卡片 → 弹 Modal → 选认证方式 → 弹窗 OAuth → 完成 | 是 |
| **非阻塞弹窗** | OAuth 在 popup 中完成，主窗口不跳转 | 是 |
| **即时反馈** | 连接成功后卡片立即变绿 | 是 |
| **设置管理** | Settings 页统一显示所有连接，可断开 | 是 |

## 二、ClaudeWiki 映射

| Rowboat 概念 | ClaudeWiki 映射 |
|-------------|----------------|
| Composio Toolkits | **Channels**（渠道：微信客服、未来可扩展飞书等） |
| OAuth popup | **一键接入 Pipeline**（QR 扫码 + Cloudflare 自动部署） |
| Connected Accounts | Settings Modal → **渠道管理** |
| ToolkitCard | **ChannelCard**（微信客服卡片） |
| ToolkitAuthModal | **ConnectWeChatModal**（连接引导 Modal） |

## 三、用户旅程

### 3.1 首次用户（未连接）

```
打开 App → Chat Tab → 看到空对话 → 发现底部"微信助手 · 未连接"
    │
    ▼
点击"微信助手" → 弹出 ConnectWeChatModal
    │
    ▼
Modal 第一步：选择连接方式
    ├── 🚀 一键接入（推荐）→ 自动部署 Cloudflare + 配置客服
    └── ⚙ 手动配置 → 已有企业微信的用户填 corpid/secret
    │
    ▼
选择"一键接入" → Pipeline 5 步执行（Modal 内展示进度）
    │
    ├── ① Cloudflare 账号 ✅ (或跳过)
    ├── ② 部署中继服务器 ✅
    ├── ③ 扫码授权企微 → 显示 QR → 等待扫码 ✅
    ├── ④ 回调 URL 配置 ✅
    └── ⑤ 创建客服账号 ✅
    │
    ▼
完成 → Modal 显示"已连接" + 客服二维码
    │
    ▼
侧边栏底部"微信助手 ●"变为绿色在线状态
```

### 3.2 已连接用户

```
侧边栏底部 → "微信助手 ● 在线" (绿点)
    │
    ├── 点击 → 弹出 ChannelStatusModal（快速状态面板）
    │   ├── 状态：在线 / 离线
    │   ├── 今日消息：收 47 / 发 42
    │   ├── 客服二维码（可分享）
    │   └── [断开连接] [查看设置]
    │
    └── 或从 Settings Modal → 渠道管理 → 详细配置
```

## 五、状态机设计

### 5.1 连接状态

```typescript
type ChannelStatus =
  | "not_connected"    // 未连接（灰色，显示"点击连接"）
  | "connecting"       // Pipeline 执行中（黄色脉冲）
  | "connected"        // 已连接，监听运行中（绿色）
  | "disconnected"     // 配置存在但监听停止（橙色）
  | "error"            // 连接错误（红色）
```

### 5.2 侧边栏底部显示逻辑

```
not_connected  → "⚡ 微信助手 · 未连接"     灰色   点击→ConnectModal
connecting     → "🔄 微信助手 · 连接中..."   黄色   点击→ConnectModal(进度)
connected      → "● 微信助手 · 在线"        绿色   点击→StatusModal
disconnected   → "○ 微信助手 · 已停止"      橙色   点击→StatusModal
error          → "⚠ 微信助手 · 异常"        红色   点击→StatusModal
```

## 六、与 Rowboat 交互模式对照

| 交互环节 | Rowboat | ClaudeWiki |
|---------|---------|-----------|
| **发现入口** | Settings → Connected Accounts 或 Tools → Toolkit Cards | 侧边栏底部"微信助手"状态行 |
| **触发连接** | 点击 ToolkitCard | 点击"微信助手 · 未连接" |
| **连接方式选择** | ToolkitAuthModal 列出 OAuth2/API Key 等 | ConnectWeChatModal 列出"一键接入"/"手动配置" |
| **认证过程** | OAuth popup 窗口 → 自动关闭 | Pipeline 进度 + QR 扫码（Modal 内） |
| **完成反馈** | 卡片变绿 + "Connected" badge | 侧边栏绿点 + 成功 Modal + 客服二维码 |
| **状态查看** | Settings → 连接表格 | 点击绿点 → StatusModal |
| **断开连接** | Settings → Disconnect 按钮 | StatusModal → 断开连接 |
| **管理配置** | Settings → Connected Accounts | Settings Modal → 微信助手 |

## 七、实施优先级

| 优先级 | 组件 | 依赖 |
|--------|------|------|
| P0 | 侧边栏底部状态行（读取 kefu status API） | 后端已有 `/status` 接口 |

| P1 | ChannelStatusModal（快速状态面板） | 后端已有 status + contact-url |
| P1 | Settings Modal 微信助手页 | 后端已有 config CRUD |

| P2 | wechat-cli 历史导入面板 | 需要新 UI 组件 |
